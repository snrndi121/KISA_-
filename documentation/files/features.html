Accordions

You can use endless accordions on one page, in multiple styles. They are easy and simple to use. Let's get started with them. 

 

Creating a Basic Accordion 

A basic accordion has an icon on the left side, a text, and a icon that animates on the right side. When this is tapped it will show the content and add an effect to the right icon. Let's explain. 

<div class="content accordion-style-2">
    <a data-accordion="accordion-content-1" href="#">
        <i class="accordion-icon-left fa fa-heart color-red1-light"></i>
        Accordion Title 1
        <i class="accordion-icon-right fa fa-plus"></i>
    </a>
    <p id="accordion-content-1" class="accordion-content bottom-10">
        Accordion Content
    </p>

    <a href="#" data-accordion="accordion-content-2">
        <i class="accordion-icon-left fa fa-star color-yellow1-dark"></i>
        Accordion Title 2
        <i class="accordion-icon-right fa fa-angle-down"></i>
    </a>
    <p id="accordion-content-2" class="accordion-content bottom-10">
        Accordion Content
    </p>

    <a href="#" data-accordion="accordion-content-3">
       <i class="accordion-icon-left fa fa-star color-yellow2-dark"></i>
       Accordion Title 3
       <i class="accordion-icon-right fa fa-arrow-down"></i>
    </a>
    <p id="accordion-content-3" class="accordion-content bottom-10">
       Accordion Content
    </p>
</div>

Line 1 generates the accordion. The accordion must be wrapped inside a content class. To create an accordion without a background use accordion-style-2 for the main class

Line 2 generates the accordion button. When you tap this it will activate the content. Notice that it uses data-accordion="accordion-content-1". The data-accordion must be a unique name for each individual accordion button.

Line 3 is the accordion left icon

Line 4 is the accordion text

Line 5 is the accordion right icon

Line 6 closes the accordion button

Line 7 is the accordion content. Notice that the id="accordion-content-1" is the same value as the data-accordion="accordion-content-1" it is very important that the accordion button and the content id are identical in name. This is how they know to find one another. Don't forget to add the accordion-content to this line. We used this with a paragraph, but you can house all the contnet inside a div. 

Line 8 is the accordion content itself.

Line 9 closes the accordion content.

To create more accordion items, simply duplicate lines 1 to 9 and replace accordion-content-1 both in the data-accordion and the id with unique names like in our code example above. 

 

Creating a Background Accordion

Let's use the code above as an example so as to not create confusion. Using the class accordion-style-2 on line 1 creates a basic accordion, we covered that. If you use accordion-style-1 that will create a set of paddings to adjust for a background accordion. So, on line 1 in the code above, we set accordion-style-1

To add the background colors, just add the class bg-VALUE-dark or bg-VALUE-light to your accordion triggers. Meaning on lines 2, 11 and 20. It's that simple! 

 

How do I create a rounded accordion?

To create a rounded accordion you must add the class accordion-toggle-first to your first accordion button and the class accordion-toggle-last to your last accordion button. In our code above, accordion-toggle-first would be added on line 2 and accordion-toggle-last would be added on line 20. 

After you add the classes you can now add the accordion round property. 

On line 1 add any of the following classes to set your accordion to be round

accordion-round-small - gives the first and last accordion toggles with a small rounded border

accordion-round-medium - gives the first and last accordion toggles with a medium rounded border

accordion-round-large - gives the first and last accordion toggles with a medium rounded border

 

Can I use other styles of accordions as well?

Yes! You can make any content an accordion. It's really simple. Use the following structure

<div class="content">
    <a href="#" data-accordion="custom-accordion-1">Accordion Trigger</a>
    <div id="custom-accordion-1" class="accordion-content">
        This is a custom accordion.
    </div>
</div>
We used this structure to create the caption accordions in the demo page. If you add data-accordion="your-unique-number" to any element and id="your-unique-number" class="accordion-content" it will automatically transform into an accordion.

Captions / Covers / Walkthroughs

With all our previous items this was the element that our customers used the most. And from your feedback we determined that you want to be able to customize captions to the extreme. That's why, with our current caption system, you can now customize each aspect of your caption. 

 

Where can I use captions?

We use captions in a tone of places! They're so versatile they can be used for anything like 

- Coverpage Sliders
- Walkthroughts
- Splash Screens
- Login / Register / Forgot Screens
- Images with Captions on Them
 
The system we built uses a single structure, that makes the code much more faster to load, less CSS and it's super easy to use. Let's explain. 

 

Caption Structure

All captions share a comon code foundation strucutre. When you want to use a caption, this is the first thing you must add. 

    <div data-height="150" class="caption caption-margins round-medium shadow-large">
        <!--Caption Type Goes Here-->
        <div class="caption-overlay bg-black opacity-90"></div>
        <div class="caption-bg bg-28"></div>
    </div>
Line 1 - Generates the caption itself. The parameter data-height="150" allows you to set the height of the caption. Previously we used responsive images for this task, but if you had text at the top and resized, it would look bad. This version allows you to easily control the height. 
 
The data-height parameter is expressed in pixels. If you want your caption to be the entire size of the screen set it to data-height="cover" . This will make it expand to the full size of the screen. 
 
Also on Line 1 we have a few secondary classes that allow us to modify the border roundness, the margins and shadow. 
 
Making Captions Rounder: use the secondary classes round-small / round-medium / round-large / round-huge
Making Captions Full Width : to make captions full width simply remove the caption-margins class
Giving Captions a Shadow: use the secondary classes shadow-small / shadow-medium / shadow-large 
 
Line 2: This is where you will place you actual content. We'll explain these bellow in Caption Type 
Line 3: Is the caption overlay, meaning, the background-color that appears over the image. You can use any secondary class for background-colors here and the opacity secondary class to reduce the background-color intensity
Line 4: Is the caption backgroudn image. You can find these in framework.css by searching for bg-28 for example. Replace the images there with your own. 
Line 5 closes the caption.
 
 
 
Caption Types
 
Inside the <!-- Caption Type Goes Here --> you can add any of the following elements. 
 
<div class="caption-center">
    I will appear in the center of the Caption
</div>

<div class="caption-top top-20 left-20">
    I will appear 20 pixels from the top and left of the Caption
</div>

<div class="caption-bottom bottom-20 left-20">
    I will appear 20 pixels from the bottom and left of the Caption
</div>
 
As you can see from their names, captions can be set at the top, bottom and center, and you can also modify how far from the top or bottom they appear. 
 
You can use 1 caption or all 3 at the same time. 
 
Inside each of the 3 caption types, caption-center / caption-bottom / caption-top you can add any element you want from our product. 
 
 
How versatile are your captions? 
Well, in all our previous products the following pages required code to position elements 
  • login and register and forgot page
  • coverpage and walkthrough 
  • coming soon and 404 pages
All these pages and their pageapps required manual code to center them. Using our captions, we can add them without any extra code. Making them much more versatile, and easier to customzie for your needs. 
 
How can I make a Cover Slider or Walkthrough Using Captions?
It's very, very easy. Let's show you an example. 
 
<div class="cover-slider owl-carousel bg-black">

    <div data-height="cover" class="caption">
        <div class="caption-center">
            <!-- Your content goes here -->
        </div>
        <div class="caption-overlay bg-black opacity-70"></div>
        <div class="caption-bg bg-28"></div>
    </div>

</div>
All you have to do, is take a caption code as seen in the first example, and use the data-height="cover" to set it to full height. Now, you wrap the entire caption in the code you see on line 1.
 
To create more cover / walkthrough slides, all you do is duplicate lines 3 to 9 in the code above, and replace your content and background image. It's that simple. 
 
 
I don't want a dark background. I want to use a CoverSlider like the Walkthrough Example. How do I do that?
It's easy. just delete the following 2 lines from any caption. 
 
    <div class="caption-overlay bg-black opacity-70"></div>
    <div class="caption-bg bg-28"></div>
By deleting these lines you are removing the overlay and the background-image, making the slider full screen and without a background image or overlay.

Content Box / Gradient Card

The content box is the most used element in our product. It serves to add a left and right margin so your content isn't stuck to the sides of the screen and automatically adds a bottom margin. 

However, that being said, the content class can be used to generate gradient boxes as well and solid color boxes too. Let's explain 

 

The basic Content Box

<div class="content SECONDARY_CLASSES">
    <!--Your Content Here-->
</div>

This is the most basic content box. It's used in all our product. It supports secondary classes to modify top, bottom, left or right margins and also to add shadows. 

 

Creating a Solid Color Content Box or a Gradient Content Box

<div class="content">
    <div class="content content-box bg-green2-dark round-medium shadow-large">
        <h4 class="color-white bold top-10">Color Box</h4>
        <p class="color-white bottom-10">
            This box is colored.
        </p>
   </div>
</div>

This will generate a content box, that is margined. Meaning, it will not expand to full width. 

Line 1 Sets the margin of the content box so it's not full width. Delete this and line 8 if you want your box to expand to full width

Line 2 uses the following classes 

- content content-box : for setting interior left and right margins.

- bg-green2-dark: is the secondary class for background color. Set it to whatever color you wish from the Secondary Classes chapter.

- round-medium: makes the box have round corners. If you plan on using the box in full width you can remove this element. You can change this to round-small / round-large / round-huge

- shadow-large: adds a shadow to the box. You can replace this with shadow-small / shadow-medium 

 

 

 

 

 

Buttons & Icons

When building buttons we always aim to give our users the most flexible and easy to use solution ever. That's why, we made the classes for buttons suit any scenario you would like to throw at them. 

If you see a button with 5 secondary classes after it, don't get scared of the amount of code there. There's nothing to worry about and it's very simple to modify. 

 

Button Class

<a href="#" class="button SECONDARY_CLASSES_HERE">Button</a>

This is the basic button structure. Every button will have the class button attached to it. From this point on, you can define your button as you need it by adding secondary classes to it. 

Button Sizes:

We've included the following sizes. Add any of the following classes to the button to change it's size.

  • button-xxs
  • button-xs 
  • button-s
  • button-m
  • button-l
  • button-xl
  • button-xxl

Button Colors

Add any bg-VALUE-light / bg-VALUE-dark from the secondary classes here replacing value with the available colors. 

Button Shape

You can change the shape of your button From square to rounded or circled by using the following classes 

  • button-round-small
  • button-round-medium
  • button-round-large
  • button-round-huge
  • button-circle

Button Position

We've included a few positioning classes to help you aling your button or increase its size to full width. 

  • button-full - increases the button to full width
  • button-center-small - centers button
  • button-center-medium - centers button and makes it medium sized in width
  • button-center-large - center button and makes it large sized in width 

Shadows

Use the secondary classes for shadows to create the button shadow, like shadow-small / shadow-medium / shadow-large

 

A real example.

Let's say you want to make a medium sized blue button, that's full width with rounded corners and a shadow added to it. Then you'd have the following code 

<a class="button button-m bg-blue1-dark button-full button-round-medium shadow-large">Button</a>

That's it! When creating a button, think about it as you were listing what you need from it. That list becomes the secondary classes you add.

Check the button component page. We added a tone of copy paste examples there.

 

What about Icons?

Icons work the same way as buttons. 

<a href="#" class="icon ICON-SIZE ICON-STYLE ICON-COLOR"><i class="fa fa-cog"></i></a>

For ICON-SIZE you can use icon-xs / icon-s / icon-m / icon-l / icon-xl / icon-xxl

For ICON-STYLE you can use icon-round / icon-square / icon-circle 

For ICON-COLOR you can use any color found in the Secondary Classes

 

 

 

 

 

Carousels

For carousels we are using the powerful OwlCarousel slider. This is a beautiful plugin with huge power and low weight. We love it and recommend using it due to its flexibility and small file size. 

For our product, there are some pre-made sliders, that you can just copy and paste and replace the content inside with your own, the best example would be the homepage slider, but you can create your own image slider using the components in the pack, especially the Captions which are highly versatile. 

 

Creating a Single Slider or a Double Slider

To create a single slide carousel, the following structure must be followed. 

<div class="slider-wrapper">
    <div class="SLIDER_TYPE owl-carousel SECONDARY_CLASS">
        <div class="item"><!--Add your content here. Duplicate to create new slide--></div>
    </div>
    <!--Controls go here-->
</div>

The above structure will generate a slider. Duplicate line 2 to create more slides. 

On line 1, replace SLIDER_TYPE with single-slider to create a single slide carousel or double-slider to create a 2 slide carousel. You can add dots under the slider by simply adding owl-no-dots to remove them or owl-dots-over to show them over the image or owl-dots-under to show the dots under the the slider.

Creating a Single Slider or Double Slider with Arrows

To add arrows to your slider, all you have to do is add the following code under your slider

<a href="#" class="prev-slide round-large bg-highlight left-20"><i class="fa fa-angle-left"></i></a>
<a href="#" class="next-slide round-large bg-highlight right-20"><i class="fa fa-angle-right"></i></a>

These arrows are generated using secondary classes, so you can change their color and position and icon with great ease.

If you are looking for advanced functionallity for our slider check out the OwlCarousel 2.3 Documentation ( https://owlcarousel2.github.io/OwlCarousel2/demos/demos.html ) - you'll find all there is to know about the slider including it's API for advanced use.

Dropdowns

Dropdowns

Dropdowns work on the same pricinple as accordions, using the data-dropdown="dropdown-1" match the id="dropdown-1" of the content to show and hide content. The difference is that accordions are dependant of on another. Meaning, if you open an accordion, then other will close.

In the case of dropdowns, they all appear as closed, and when you tap one it will open. Tapping another one will open that as well while not closing the previous one.

<div class="content">
    <a href="#" data-dropdown="dropdown-1" class="dropdown-style-1 SECONDARY_CLASS">
        <i class="fa fa-share-alt color-red1-light"></i>
        <p class="color-theme">Social</p>
        <i class="fa dropdown-icon fa-angle-down color-theme"></i>
    </a>

    <div class="dropdown-content" id="dropdown-1">
        <!--Dropdown Content Here-->
    </div>
</div>

If you wish to use colored accordions simply replace dropdown-style-1 with dropdown-style-2 and replace SECONDARY_CLASS with a background color of your choosing. 

 

 

 

Header & Logos

Our products come with a powerful header that will allow you to choose it's desing by simply replacing one class. Headers support text logos and image logos and these are replaced with incredible ease. 

 

Let's first explain the header styles and then we'll explain where you can change your logo. If you want to skip this step, just scroll down a bit. 

<div class="header HEADER-POSITION HEADER-STYLE">
   <a href="#" class="header-title">Fonts</a>
   <a href="#" class="header-icon header-icon-1"><i class="fas fa-arrow-left"></i></a>
   <a href="#" class="header-icon header-icon-2"><i class="fas fa-bars"></i></a>
   <a href="#" class="header-icon header-icon-3"><i class="fas fa-cog"></i></a>
   <a href="#" class="header-icon header-icon-4"><i class="fas fa-moon"></i></a>
</div>

This is the header. It supports a tone of styles, so let's explain each of them

Line 1 has the main header class.

Replace HEADER-POSITION with:

header-fixed - for a fixed header that will stay sticky at the top.

header-static - for a static header that will scroll with the page. NOTE if you want a static header you will have to move the header inside the page-content class. Currently, the header is always placed outside the page-content class.

Replace HEADER-STYLE with:

header-logo-app will have an icon, the logo or title and 3 icons on the right

header-logo-left will have the logo at the left and 4 icons on the right

header-logo-center will have the logo in the center and 2 icons on the left side and 2 icons on the right

header-logo-right will have 4 icons on the left and the logo on the right.

Line 2 is the header title. You can replace header-title with header-logo to use a logo image instead of the text. 

Lines 3, 4, 5, 6 are the header icons. These have the class header-icon and header-icon-1 / header-icon-2 / header-icon-3 or header-icon-4. The icons will adapt depending on the HEADER-STYLE

 

How do I make the header transparent, like the coverpage and walkthroughts?

Just add the class header-transaprent to your header. And the header will automatically be transaprent.

 

How do I set my own logo?

It's very simple. Just open style.css and at the very top you'll see all the images for the logos. Replace the image there with your own and set the background-size to half of your logo to maintin retina aspect. So if your logo is 100x50 pixels as an image file you will set background-size:50px 25px. 

The same principle applies for the menu where you can change nav-title with nav-logo. and the footer where you can relace footer-title with footer-logo

Snackbars

Shackbars are small notifications that show up from the bottom of the page. These can be activated manually, by adding them above the page-content class or triggered automatically by adding them to any button or link you want. 

Automatic snackbars are useful for callbacks, when you want a message to show when you tap a button. 

 

On any page you have snackbars, outside the page-content you must add the following code

    <div class="snackbars snackbars-boxed snackbar-round">
        <!-- Only manual snackbars get added here-->
    </div>
Automatic Snackbars

Let's say you have a link, that you want to show a snackbar when you tap. Then, we'd use the following code

                <a href="#"
                   data-snack-id="snack-1"
                   data-snack-color="bg-highlight color-white"
                   data-snack-text="I'm the color of the page highlight!"
                   data-snack-icon="fa fa-check">
                       I will show a SnackBar
                </a>
Line 1 opens the anchor
Line 2 is a unique identifier for your snackbar. Each snackbar must have a unique number here
Line 3 is the color. You can use the secondary class names here
Line 4 is the text you want the snackbar to show
Line 5 is the icon that will appear on the right side of your snackbar
Line 6 is the link text. You will only see this element on screen before tapping it
Line 7 closes the snackbar. 
 
 
Manual Snackbars.
Manual snackbars must be placed outside the page-content class.  The following code must be added inside the box mentioned earlier. 
 
 
<a href="#" class="bg-blue2-dark" id="snackbar-manual-1"><i class="fa fa-sync fa-spin"></i>Just a second...</a>
This code will be your snackbar. Notice it has the id="snackbar-manual-1" this id must be unique and corespond to the data-snack-manual-id="manual-1" to show the snackbar. It works on the same principle as the dropdowns and accordions. 
 
 
 
 
 
 
 
 
 

Tabs

Tabs are very simple to work. They rely on the same principle as the accordions, dropdowns and snackbars, using data attribute to match an ID. Let's explain 

<div class="content">
    <div class="tab-controls tab-animated tabs-small tabs-rounded"
         data-tab-items="4"
         data-tab-active="bg-highlight">

        <a href="#" data-tab-active data-tab="tab-1">Tab 1</a>
        <a href="#" data-tab="tab-2">Tab 2</a>
        <a href="#" data-tab="tab-3">Tab 3</a>
        <a href="#" data-tab="tab-4">Tab 4</a>
    </div>
    <div class="clear"></div>

    <div class="tab-content" id="tab-1">
        <!--Tab Content 1-->
    </div>

    <div class="tab-content" id="tab-2">
        <!--Tab Content 2-->
    </div>

    <div class="tab-content" id="tab-3">
        <!--Tab Content 3-->
    </div>

    <div class="tab-content" id="tab-4">
        <!--Tab Content 4-->
    </div>
</div>

Line 1 wrapps the tabs inside a content box. This is required.

Line 2 is the tab controls housing box. This has the mandatory tab-controls and tab-animated clases which are used by javascript and you can replace tabs-small with tabs-medium or tabs-large to change the size of the tabs and also, if you remove tabs-rounded they will become square.

Line 3 is the number of tabs you want to have on. We're using 4 in this example

Line 4 is the background color of the selected tab. We're using highlight in this example but you can use any color you want from the secondary classes. 

Line 6,7,8,9, are the tab buttons that get tapped to change the tab. add  data-tab-active to the tab you want to have activated by default. 

Notice that all of these have data-tab="tab-NUMBER" that match with the id="tab-NUMBER" You can create however many tabs you wish, just remember that these will appear on a small screen and that the must have the same data-tab and id.

Toggles

One of the biggest requests our customers had were mobile styled toggles. And these always required fine tuning for each customer preference, so we made the flexible to the point where you can design it to fit whatever need you have. 

 

Toggles have 2 main styles. iOS Design and Android Design. Let's explain 

 

iOS Design

    <a href="#" class="toggle-switch toggle-ios toggle-off"
       data-toggle-height="30"
       data-toggle-width="55"
       data-toggle-content="toggle-content-1"
       data-toggle-checkbox="toggle-checkbox-1"
       data-icons-size="9"
       data-bg-on="bg-green1-dark"
       data-bg-off="bg-red2-dark">
        <span class="color-theme regularbold font-14">iOS Toggle Style 2</span>
        <strong></strong>
        <i class="fa-t1 fa fa-check"></i>
        <i class="fa-t2 fa fa-times"></i>
        <u></u>
    </a>
    <input type="checkbox" id="toggle-checkbox-1">
    <div class="toggle-content" id="toggle-content-1">
        <!--This will be your toggle content-->
    </div>
 
Let's explain line by line. 
 
Line 1. Is the toggle itself. You can replace toggle-off with toggle-on to have it activated by default
Line 2 is the toggle height. Value in pixels
Line 3 is the toggle width. Value in pixels.
Line 4 is the id that you want to show. This value will be added to the toggle-content class at the bottom of the code
Line 5 is the checkbox you want to tick. This value will be added the input type="checkbox" at the bottom of the code.
Line 6 is the icon font size in case you want to use icons under the buttons
Line 7 is the background color when the toggle is active
Line 8 is the background color when the toggle is inactive
Line 9 is the text that will appear on the toggle. 
Line 10 is the ball of the toggle. Leave the strong there empty as shown
Line 11 is the icon for the active state
Line 12 is the icon for the inactive state
Line 13 is the background of the toggle
Line 14 closes the toggle. 
 
That's it. You can change all these values to use the toggle with any design you want. If you don't wish to use icons in the toggle, simply delete lines 11 and 12. 
 
Line 15 is the checkbox for the toggle. If you plan on using the toggle as checkbox, add this value here. If you want the checkbox to be invisible just add the class opacity-0 to this checkbox. 
Line 16 is the content for the toggle. This will show and hide depending on the toggle. This must have the class toggle-checkbox. 
 
Both Lines 15 and Lines 16 must have the same id="toggle-checkbox-NUMBER" or "id="toggle-content-NUMBER" as the values you add in line 4 and 5. Otherwise they will not work.
